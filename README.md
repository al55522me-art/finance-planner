#Отчет

Постановка задачи
Цель проекта — разработать консольное приложение на языке Python для ведения учёта личных финансов. Программа должна позволять пользователю добавлять доходы и расходы, распределять траты по категориям, просматривать баланс и формировать отчёты за выбранный период.
В рамках проекта необходимо реализовать следующий функционал:
	•	добавление доходов;
	•	добавление расходов;
	•	указание категории операции;
	•	просмотр списка всех операций;
	•	подсчёт общего дохода, расходов и остатка средств;
	•	формирование отчёта по категориям;
	•	формирование отчёта за выбранный период;
	•	хранение данных в файловом формате.
Интерфейс приложения должен быть реализован в консольном виде и быть понятным для пользователя. Пользователь должен иметь возможность выбирать действия из меню и получать результат без сложных дополнительных настроек.

Анализ задачи
Для решения задачи выбрана объектно-ориентированная парадигма, так как она естественно подходит для предметной области финансового учёта. В программе присутствуют сущности, которые удобно описывать отдельными классами: операция, категория, доход, расход, менеджер финансов и генератор отчётов.
Основной принцип проектирования — разделение ответственности:
	•	классы моделей хранят данные;
	•	класс менеджера отвечает за добавление и обработку операций;
	•	класс отчётов формирует текстовые сводки;
	•	консольный интерфейс обеспечивает взаимодействие с пользователем.
Для хранения данных используется файловый формат, например JSON. Такой формат подходит, потому что:
	•	легко читается и записывается;
	•	поддерживает структуру данных;
	•	удобен для хранения списка операций;
	•	может быть использован для последующего расширения проекта.
Обоснование выбора ООП
В проекте реализуются следующие принципы ООП:
	•	Инкапсуляция — данные и логика обработки операций находятся внутри классов.
	•	Наследование — доход и расход можно представить как частные случаи базовой сущности операции.
	•	Полиморфизм — разные типы операций можно обрабатывать единым способом, отличая их только по типу.
Таким образом, объектно-ориентированный подход позволяет сделать программу расширяемой, понятной и удобной для сопровождения.

Структура классов
1.  Category 
Класс категории финансовой операции.
Назначение:
Хранит название категории, например: «Еда», «Транспорт», «Обучение».
Поля:
	•	 name  — название категории.

3.  Transaction 
Базовый класс финансовой операции.
Назначение:
Служит общей моделью для всех операций. Содержит сумму, категорию, описание, дату и тип операции.
Поля:
	•	 amount  — сумма операции;
	•	 category  — объект категории;
	•	 description  — описание;
	•	 date  — дата и время операции;
	•	 transaction_type  — тип операции.

Особенности:
В классе реализована проверка корректности суммы, чтобы исключить отрицательные и нулевые значения.
3.  Income 
Класс дохода, наследуется от  Transaction .
Назначение:
Представляет поступление денежных средств.
Особенности:
При создании объект автоматически получает тип  income.
4.  Expense 
Класс расхода, наследуется от  Transaction.
Назначение:
Представляет списание денежных средств.
Особенности:
При создании объект автоматически получает тип  expense.

5.  FinanceManager 
Основной управляющий класс приложения.
Назначение:
Отвечает за хранение всех операций и выполнение основных вычислений.
Функции:
	•	добавление доходов;
	•	добавление расходов;
	•	получение списка всех операций;
	•	фильтрация по типу;
	•	фильтрация по категории;
	•	фильтрация по периоду;
	•	вычисление суммы доходов;
	•	вычисление суммы расходов;
	•	вычисление баланса.
Роль в проекте:
Является центральным классом, через который проходит вся работа с финансовыми данными.

6.  ReportGenerator 
Класс формирования отчётов.
Назначение:
Создаёт текстовые отчёты для пользователя.
Функции:
	•	общий финансовый отчёт;
	•	отчёт за период;
	•	отчёт по категориям.
Роль в проекте:
Позволяет представить данные в удобном для пользователя виде.
7. Консольный интерфейс
Отдельный модуль пользовательского взаимодействия.
Назначение:
Обеспечивает меню, ввод данных и вывод результатов.
Основные действия пользователя:
	•	добавить доход;
	•	добавить расход;
	•	посмотреть баланс;
	•	вывести отчёт;
	•	выйти из программы.

Вывод
В ходе выполнения проекта была разработана консольная программа «Планировщик личных финансов» на языке Python в объектно-ориентированном стиле. Реализованы основные функции учёта доходов и расходов, а также построения отчётов. Использование ООП позволило логично разделить сущности и упростить дальнейшее расширение программы.

# Personal Finance Planner

Консольное приложение на Python для учёта личных финансов.  
Проект позволяет добавлять доходы и расходы, распределять траты по категориям и формировать отчёты.

## Возможности

- Добавление доходов.
- Добавление расходов.
- Категоризация операций.
- Просмотр всех транзакций.
- Подсчёт баланса.
- Отчёт по категориям.
- Отчёт за выбранный период.

## Стек технологий

- Python 3.x
- ООП
- datetime
- dataclasses
- enum
- typing
